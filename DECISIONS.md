# 결정 로그 (Decision Log)

이 문서는 Recipe Book 웹앱의 주요 디자인·UX·구현 결정을 **배경(왜 문제였나) → 결정(무엇을 했나) → 이유(왜 그렇게)** 순서로 기록한다.
앞으로 의미 있는 변경은 커밋 메시지에 이유를 남기고, 이 문서에도 항목을 추가한다.

관련 좌표/수치는 기준 뷰포트 **390 × 844 CSS px**에서 측정한 값이다.

---

## 2026-07-24

### 1. 타이틀 워드마크는 GPT 시안을 그대로 벡터/PNG로 추출해 사용
- **배경**: "My recipe book" 타이틀을 웹폰트로 흉내 내면 시안의 세리프 디테일(스와시, 자간)이 재현되지 않는다.
- **결정**: 레퍼런스 이미지에서 글자 영역을 잘라 **알파 키(luma 기반)** 로 투명 PNG를 추출해 배경 워터마크·스플래시·아이콘에 공통으로 사용한다.
- **이유**: 사용자 요구가 "다른 폰트를 찾지 말고 시안 그대로"였고, 오프라인 PWA라 외부 폰트 로드도 피해야 한다. 단일 PNG를 공유하면 톤이 일관된다.
- **추출 파라미터**: `a = clamp((luma - 48) / 46, 0, 1)`. 밑줄 포함본은 크롭 하단을 y≈613까지 넓혀 원형 끝점까지 온전히 담는다.

### 2. 장식 밑줄 복원 및 또렷하게 재드로우
- **배경**: 초기 추출본은 밑줄을 y=600에서 잘라 원형 끝점이 반쪽만 들어가 "깨진 점"처럼 보였다. 이후 사용자는 삭제가 아니라 **온전한 밑줄 복원**을 원했다.
- **결정**: (a) 크롭을 넓혀 밑줄+양끝 원을 통째로 추출. (b) 흐린 원본 밑줄 대신, 글자색(밝은 레드 `rgb(229,59,54)`)에 맞춰 선을 약간 두껍게 + 링 끝점을 또렷하게 다시 그렸다.
- **이유**: 잔점은 크롭 경계가 밑줄을 관통해서 생긴 아티팩트였다. 온전히 담으면 아티팩트가 사라지고, 배경 워터마크(투명도 0.22)에서도 선이 읽히려면 대비를 올려야 한다.

### 3. 스플래시 하단 세로 태그라인 (CAPTURE · PARSE · SCALE · EVOLVE)
- **배경**: 사용자가 시안 이미지를 첨부하며 스플래시에 태그라인을 요청.
- **결정**: 시안에서 **글자를 직접 추출**해 세로로 스택, 단어 사이에 원형 점. 간격은 시안 실측값(글자 높이≈78px, 원소 간 여백≈120px, 점 지름≈24px)을 그대로 사용. 색은 약간 밝게, 알파 강화. 크기는 높이 기준 60%로 축소. 가로 중심을 **배경 장식선의 중심(뷰포트 x축 약 44% 지점)** 에 맞춤.
- **이유**: 가로 배열은 시안과 달라 거부됨 → 세로 유지. 장식선 중심이 스와시 때문에 뷰포트 중앙보다 왼쪽이라, 태그라인 점열을 그 중심에 맞춰야 시각적으로 정렬돼 보인다. 태그라인은 스플래시 전용(아이콘·배경 미적용).

### 4. 앱 아이콘: 타이틀 80% 축소 + 잔점 제거
- **배경**: 홈 아이콘의 글자가 크게 느껴졌고, 추출 원본의 밑줄 잔점이 아이콘 하단에 보였다.
- **결정**: 아이콘용 타이틀은 밑줄이 없는 **깨끗한 크롭**(하단 y≈592, 밑줄 위에서 컷)을 쓰고, 스케일을 기존 0.95의 80%인 **0.76**으로. 버건디 `#550010` 배경에 중앙+미세 보정 배치.
- **이유**: 아이콘엔 밑줄이 불필요하고, 밑줄을 배제하면 잔점 문제도 원천 제거된다.

### 5. 리스트 상단 그림자: 점진 페이드인 + 장식선에 위치 고정
- **배경**: 그림자가 스크롤 시작 즉시(`scrollTop > 8`) 튀어나와 과했다.
- **결정**: 불투명도를 스크롤량에 비례시킨다 — `op = clamp((scrollTop - 12) / 36, 0, 1)` (≈12px까진 0, ≈48px에서 1). 상태를 boolean `scrolled` → 수치 `shadowOp`로 교체.
- **이유**: 첫 항목이 장식선 아래로 들어가기 시작하는 지점(≈27px 스크롤) 즈음부터 서서히 나타나야 자연스럽다. "너무 빨리 나오지 않고 거의 다다랐을 때."

### 6. 리스트 잘림(clip) 지점을 그림자선(=장식선, 244px)과 일치
- **배경**: 그림자선은 장식선(244px)에 있는데, 리스트 오버플로 잘림 경계는 컨테이너 상단(214px)이라 **30px 어긋나** 항목이 그림자보다 위에서 잘렸다. → "그림자 아래로 사라져야 하는데 안 맞는다."
- **결정**: 리스트 컨테이너 상단(`listTop`)을 장식선 y와 동일한 **244px**로 내리고, 상단 그림자를 box-shadow 대신 **아래 방향 전용 선형 그라데이션**으로 교체해 잘림선에 핀 고정. padding-top을 줄여(57→34) 첫 행 위치는 유지.
- **이유**: 잘림 경계와 그림자를 같은 y로 두면 항목이 장식선에서 정확히 잘려 그 아래로 사라진다. 그라데이션은 위쪽 번짐이 없어 검색창~리스트 사이 여백을 오염시키지 않는다.

### 7. 헤더 간격 조정 시 리스트 잘림선(244px)은 불변 유지
- **배경**: 장식선은 배경 워터마크(고정 300px 폭, top −2px)라 **y≈244px에 항상 고정**이다. 탭·검색창을 위아래로 옮기면 그 아래 리스트가 딸려 움직여 244 정렬이 깨진다.
- **결정**: 한 요소만 옮길 땐 **margin-top/margin-bottom을 같은 양만큼 반대로** 조절해(예: 검색창 ↓30 = margin-top +30 / margin-bottom −30) 아래 흐름을 상쇄하고 `listTop`을 244로 유지한다. 요소 이동 후엔 항상 `listTop`을 재측정해 확인.
- **이유**: 그림자·잘림 정렬(항목 6)을 깨지 않으면서 개별 요소만 재배치하기 위한 불변식.
- **주의**: "버튼 ↑ + 검색창 ↓"를 같은 값으로 하면 서로 상쇄돼 검색창이 제자리처럼 보인다(버튼을 올리면 아래 검색창도 딸려 올라감). 검색창만 움직이려면 그 요소의 margin으로 처리.

### 8. PWA / 서비스워커: HTML은 network-first, 에셋은 cache-first
- **배경**: 초기 SW가 HTML을 cache-first로 서빙해 배포 후에도 옛 화면이 붙어 있었다.
- **결정**: navigation(HTML)은 network-first(+캐시 갱신), 그 외 에셋은 cache-first. 프리캐시 목록이 바뀌면 `CACHE` 버전을 올린다(현재 `recipe-book-v3`).
- **이유**: 업데이트가 즉시 반영되면서 오프라인 fallback도 유지된다. 새 에셋(예: `tagline.png`) 추가 시 버전을 올려 강제 갱신.

### 9. 배포 절차 (standing instruction)
- **배경**: 사용자가 배포 전 미리보기 확인을 원함("배포전에 나랑 확인해").
- **결정**: 변경 후 **프리뷰 스크린샷을 먼저 공유**하고, 사용자가 "배포"라고 할 때만 `main`에 커밋·푸시한다(푸시 시 GitHub Pages 자동 배포). github.io 직접 접속이 막힌 환경이라 배포 성공은 GitHub Actions 실행 상태로 확인.
- **이유**: 픽셀 단위 튜닝이 잦아 반복 확인이 안전하고, 의도치 않은 배포를 막는다.
